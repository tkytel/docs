GitHub の issue 及び pull request の運用方法について

1. 概要

GitHub のレポジトリにおける同時編集を防ぐため、次のように案内します:

- issue を用いたチケット発行を強く推奨します
- issue に assign された人は、なるべく早い段階で少なくとも branch を push し、また、issue に関連付けられた draft PR を作成してください
  - コミットは乱雑で良いです（小さければ小さいほど良いです）
  - これがなされない場合、いわゆる ***やるやる詐欺*** として別の人が issue に取り組む可能性があります
- ほどよく進捗したら、draft PR を PR に変更して review してもらってください
  - 誰に review してもらえば良いかわからない場合、そのレポジトリの最初のコミットを作った人か、@kusaremkn または @yude に依頼すれば良いです
- Review を受けて、編集したりしなかったりしてください
  - 別の人がその branch に push することもあります
- ほどよく（おだやかに）
